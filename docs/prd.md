# PRD — AI Children’s Book Platform

## Problem
Parents/educators want personalized children’s stories, but current AI workflows struggle with:
- **Character consistency** across pages
- Time-consuming layout/export
- Unreliable safety filters

## Vision
A guided, safe platform that turns a story idea into a **print-ready illustrated book + narrated audiobook** in minutes, with characters that stay consistent across the entire book and across a series.

## Target users
- Busy parents (primary)
- Educators (secondary)
- Children’s content creators / prosumers (secondary)

## Core user outcomes
- Produce a kid-safe story matching age, interests, and moral.
- Keep characters visually consistent page-to-page.
- Export as PDF + MP3/M4B.

## MVP features
1. **Book wizard**: prompt → age band → length/pages → style preset → characters.
2. **Storyboard**: page-by-page text with scene descriptions.
3. **Character bible**: canonical character designs + traits + palette.
4. **Illustration generation**: per page (consistent characters) with regen.
5. **Layout templates**: 2–3 templates; typography presets.
6. **Narration**: voice selection; per-page or full-story script; generate audio.
7. **Exports**: PDF + audio bundle.
8. **Safety**: prompt moderation + output checks; banned topic filters.

## Premium (post-MVP)
- Motion-lite video (pans/parallax) and/or chapter clips via video gen.
- Series engine (shared universe).
- Multi-language.

## Success metrics
- Time-to-first-book < 10 minutes (MVP)
- Regen rate per page <= 2 on average
- NPS from parents/teachers
- CAC payback < 2 months on subscription tiers

## Key constraints
- No default collection of child biometrics.
- Moderation required before sharing.
